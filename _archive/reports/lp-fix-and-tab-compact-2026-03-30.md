## 问题背景 / Problem Background

Boss 提出两个问题：

1. **生活压力评分反直觉**: 慕尼黑从"节省"切换到"适中"消费档位时，生活压力指数反而从 13 降到了 9（更低 = 压力更小），这显然不合理——花更多钱应该压力更大。
2. **排行榜标签栏布局**: 5行分组标签选择器（收支/住房/工作/环境/指标），每行按钮集中在左侧，右侧大量空白。

## 迭代讨论记录 / Iteration Chat Log

### 第一轮：问题分析

**🔧 [16:30] 工程师**: 先分析问题一。当前 `computeLifePressure()` 使用 `minMaxNorm()`，即在所有城市的数据集上做 min-max 归一化。这是相对评分，问题在于：
- 切换消费档位时，所有城市的储蓄率、购房年限都同时变化
- 整个数据集的 min/max 边界随之移动
- 某些城市的相对排位可能反直觉变化

我用 Python 模拟了一下，确认了 bug：慕尼黑在 Budget 模式下 LP=5，Moderate 模式下 LP=3，确实降低了。

**🧪 [16:33] 测试员**: 根本原因很清楚——`minMaxNorm` 将整个数据集压缩到 [0, 100]，切换到 moderate 后所有城市储蓄率区间从大约 [0.32, 0.93] 压缩到 [0.18, 0.90]，慕尼黑在新的相对尺度上反而"不那么差"了。这不是一个城市的问题，是系统性问题。

**🃏 [16:35] 捣蛋鬼**: 那干脆不做归一化了？直接用原始数据排名不就完了？

**🔧 [16:37] 工程师 [否决]**: 不行，归一化是为了把不同量纲的子指标（储蓄率 0-1、巨无霸购买力 1-20、工作时长 1200-2500、购房年限 3-100+）混合加权。没有归一化就无法做加权求和。

**🎨 [16:38] 设计师**: 安全指数、医疗指数、自由指数有同样的问题吗？

**🔧 [16:39] 工程师**: 没有。Safety/Healthcare/Freedom 指数是在 `cities.json` 中预计算好的绝对值，不依赖用户选择的消费档位或职业。只有 Life Pressure Index 是客户端实时计算的，受用户设置影响。

### 第二轮：方案讨论

**🔧 [16:42] 工程师**: 提出方案——将 `minMaxNorm` 替换为「固定锚点归一化」（anchoredNorm）。用预设的、从数据集极值推导的固定区间做归一化：
- 储蓄率: -0.30 (最差) … 0.85 (最好)
- 巨无霸购买力: 1.0 … 22.0
- 年工作时长: 2500 (最差) … 1200 (最好)
- 购房年限: 120 (最差) … 3 (最好)

超出范围的值会被 clamp。这样无论用户选什么档位/职业，同一个原始值永远映射到同一个 0-100 分数。

**🧪 [16:45] 测试员**: 锚点值怎么确定的？

**🔧 [16:46] 工程师**: 从当前 134 城市数据集的极值出发，加合理安全裕度：
- 储蓄率最低约 -0.25（部分城市收不抵支），取 -0.30；最高约 0.82，取 0.85
- 巨无霸购买力最低约 1.2，取 1.0；最高约 20，取 22.0
- 工时最低约 1380，取 1200；最高约 2440，取 2500
- 购房年限最低约 4，取 3；极高值 clamp 到 120

**🃏 [16:48] 捣蛋鬼**: 未来加新城市如果超出锚点范围怎么办？

**🔧 [16:49] 工程师**: 会被 clamp 到 0 或 100，即"极端好"或"极端差"。这比 minMaxNorm 好——minMaxNorm 加一个极端城市会影响所有城市的分数，而固定锚点只影响新城市自己（被 clamp），其他城市分数不变。真需要更新锚点时可以定期审查。

**🎨 [16:51] 设计师**: 函数签名需要改吗？会影响调用方吗？

**🔧 [16:52] 工程师**: 保持原签名不变。`allCities` 和 `allIncomes` 参数加 `_` 前缀标记为未使用，但不删除，避免修改所有调用方。零破坏性变更。

### 技术细节

**anchoredNorm 函数**:
```typescript
function anchoredNorm(lo: number, hi: number, val: number): number {
  if (hi === lo) return 50;
  const clamped = Math.max(lo, Math.min(hi, val));
  return ((clamped - lo) / (hi - lo)) * 100;
}
```

**固定锚点参数**:
| 子指标 | 锚点下限 (最差) | 锚点上限 (最好) | 权重 |
|---|---|---|---|
| 储蓄率 | -0.30 | 0.85 | 30% |
| 巨无霸购买力 | 1.0 | 22.0 | 25% |
| 年工作时长 | 2500h | 1200h | 25% |
| 购房年限 | 120年 | 3年 | 20% |

**标签栏布局对比**:
- 旧: 5行 `space-y-2`，每行 `w-12` 分组名 + 按钮
- 新: 单行 `flex flex-wrap gap-x-1 gap-y-1.5`，组间 `|` 分隔

## 关键决策 / Key Decisions

1. **固定锚点 vs 动态归一化**: 选择固定锚点，牺牲极少量的灵活性换取跨档位/跨职业的绝对可比性
2. **保持函数签名**: `allCities`/`allIncomes` 参数保留但标记未使用，零破坏性变更
3. **去掉分组名标签**: 按钮 emoji 前缀已足够区分分组，`|` 分隔线更节省空间
4. **不做下拉菜单**: 16个选项的平铺按钮比下拉菜单更易扫视和操作
