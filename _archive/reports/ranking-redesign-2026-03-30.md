## 迭代讨论记录 / Iteration Chat Log

### 第一轮：需求分析与架构决策

**🔧 [10:00] 工程师**: 当前排行榜页（419行）存在以下问题：
1. 13个标签页，不包含全部16项数据（缺少年收入、月支出、房价、时薪）
2. 硬编码颜色阈值（如AQI≤50为绿色），不同于城市详情页的百分位制（前20%/后20%）
3. 自行管理状态而非使用 useSettings hook，与其他页面不一致
4. 表格列定义重复冗余

**🎨 [10:05] 设计师**: 视觉一致性是关键。城市详情页将数据按4个分组展示（收支/住房/工作/环境+指标），排行榜标签页也应分组。建议：
- 标签按5组排列（收支/住房/工作/环境/指标），每组前有标签名
- 同组非活跃标签使用浅色高亮区分
- 值颜色使用百分位制着色（同城市详情页：前20%绿、后20%红）

**🧪 [10:10] 测试员**: 关注点：
1. 16个标签在移动端会拥挤→需要flex-wrap滚动
2. null值处理：许多城市某些指标为null
3. 指标子排序：切换指标标签时需重置子排序
4. Hook顺序：early return不能在hooks之前

**🃏 [10:15] 捣蛋鬼**: 何不给每个单元格加进度条可视化？

**🔧 [10:18] 工程师 [否决]**: 134行×多列的进度条会造成视觉噪音。保持简洁的tier coloring方案。

### 第二轮：标签分组与列设计

**🔧 [10:20] 工程师**: 确定标签分组方案：
- 收支 (3): 💰年收入 | 💸月支出 | 📈年存款（原💰存钱排行）
- 住房 (3): 🏘️房价 | 🏠购房年限 | 🏠房租
- 工作 (3): ⏰工作时长 | 💵时薪 | 🏖️年假
- 环境 (3): 🌿空气 | 🌐网速 | ✈️航班
- 指标 (4): 😰生活压力 | 🛡️安全 | 🏥医疗 | 🗽自由

**🎨 [10:25] 设计师**: 点击某组内的标签时，表格显示该组全部3列（指标组显示指标+子数据列）。列头可点击切换排序。指标标签页的子数据列头显示权重百分比。

**🧪 [10:28] 测试员**: 子排序方向问题——每个子指标都有不同的"好"方向：
- 生活压力子指标：储蓄率↑、巨无霸购买力↑、工时↓、购房年限↓
- 安全子指标：全部↑（已经反转处理过的数据）
- 医疗/自由子指标：全部↑

**🔧 [10:30] 工程师**: 在排序逻辑中逐项配置方向。用nullLast函数统一处理null值排序。

### 第三轮：技术实现

**🔧 [10:35] 工程师**: 核心重构决策：
1. 使用 useSettings hook 替代手动 localStorage 管理
2. useMemo 优化所有计算（allIncomes, rows, V, sorted）
3. 定义 TC 组件统一处理tier着色
4. 定义 SubTh 组件处理可点击的指标列头
5. renderHeaders + renderCells 按分组渲染

**🧪 [10:40] 测试员**: Hook调用顺序——`if (!s.ready) return null` 必须放在所有hooks之后。否则ESLint报错"hooks called conditionally"。

**🔧 [10:42] 工程师**: 将 early return 移到 JSX return 之前，所有 hooks 之后。

**🎨 [10:45] 设计师**: 表格头部和行样式保持与之前一致——前3名高亮、奖牌emoji、城市可点击跳转。数据列用百分位tier着色。低置信度数据仍标 * 号。

**🃏 [10:48] 捣蛋鬼**: 非指标分组的列头也应可点击切换tab——等于点了另一个排序标签。

**🔧 [10:50] 工程师**: 好主意，已采纳。非指标分组列头点击 → handleTab(gTab) → 切换到该标签。指标分组列头点击 → handleSubSort(sk) → 切换子排序。

### 第四轮：i18n与构建

**🔧 [11:00] 工程师**: 新增i18n keys（4个语言×9个keys = 36条翻译）：
- 4个新标签: rankTab_income, rankTab_expense, rankTab_housePrice, rankTab_hourlyWage
- 5个分组: rankGroup_income, rankGroup_housing, rankGroup_work, rankGroup_environment, rankGroup_index

**🧪 [11:05] 测试员**: 构建通过，220页全部生成。排行榜页体积从旧版的多个冗余列逻辑简化为配置驱动。

## 改动摘要 / Changes Summary

### 文件变更

| 文件 | 改动 |
|---|---|
| `components/RankingContent.tsx` | 完全重写（419行→504行） |
| `lib/i18n.ts` | +36行（4语言×9新键） |

### 架构对比

| 项目 | 旧版 | 新版 |
|---|---|---|
| 标签数 | 13 | 16 |
| PPP标签 | ✅ 有 | ❌ 移除（不在16项数据中） |
| 状态管理 | 手动useState+localStorage | useSettings hook |
| 颜色逻辑 | 硬编码阈值 | 百分位制（前20%/后20%） |
| 标签布局 | 平铺网格 | 分组（5组+标签） |
| 列数据 | 每标签仅1-4列 | 每组3列/指标5列 |
| 指标子排序 | ❌ 无 | ✅ 可点击列头排序子数据 |
| 列头交互 | 不可点击 | 全部可点击（切换排序/切换标签） |

### 新增功能

1. **分组标签选择器**: 5个分组（收支/住房/工作/环境/指标），每组有group label
2. **16项完整数据**: 新增年收入、月支出、房价/m²、时薪4个标签
3. **指标子数据排序**: 点击指标的子数据列头改变排序依据
4. **统一百分位着色**: 使用rankHigher/rankLower+tierHigh/tierLow，与城市详情页完全一致
5. **同组标签高亮**: 当前分组的非活跃标签使用浅色高亮

### 移除项

- 购买力(PPP)标签：不在城市详情页16项数据中
- 硬编码颜色阈值：统一改为百分位制
- 手动localStorage管理：改用useSettings hook
