## Bug 1：排行榜年收入 emoji 无法显示

**根因**: 上次修改将 💰 替换为 📈 时，文件编码损坏。📈 (U+1F4C8, UTF-8: `f0 9f 93 88`) 被写为 U+FFFD 替换字符 (UTF-8: `ef bf bd`)，导致浏览器显示 `�`。

**修复**: 使用 Python 脚本精确定位并替换 4 字节序列，4 个语言版本全部修复。

**修改文件**: `lib/i18n.ts` (4 处)

---

## Bug 2：排行榜和对比页顶栏挡住表格内容

**根因分析（团队讨论焦点）**:

顶栏使用 `sticky top-0 z-50`，当用户滚动页面时，顶栏固定在视口顶部不动。同时，表格 `<thead>` 试图使用 `sticky top-[49px] z-40` 吸顶，但由于表格被嵌套在 `overflow-hidden` 和 `overflow-x-auto` 容器中，CSS 规范规定 `position: sticky` 无法穿透 overflow 非 visible 的祖先容器——因此 thead 吸顶完全失效。

结果：用户滚动时只看到固定的顶栏 + 没有表头的表格数据，产生"顶栏挡住内容"的体验。

**团队讨论要点**:

- 工程师: `sticky` + `overflow-hidden/auto` 是 CSS 已知的不兼容组合，无法在当前 DOM 结构下同时实现水平滚动和垂直吸顶
- 设计师: 顶栏不需要始终悬浮，用户翻阅 134 行数据时需要的是表头而非导航栏
- 找茬的: 很多数据网站顶栏都不是 sticky 的，sticky 反而会让表格看起来被"压"了一层
- 测试员: 确认在 375px 移动端宽度下，顶栏 flex-wrap 后高度超过 49px，即使 sticky 生效也会和 thead 重叠

**修复**:
- 排行榜: 移除顶栏 `sticky top-0 z-50`，移除 thead `sticky top-[49px] z-40`
- 对比页: 同样处理

**修改文件**: `components/RankingContent.tsx`, `components/CompareContent.tsx`

## Bug 3：对比页新增城市下拉框被截断

**根因**: "+"按钮及其下拉框处于 `overflow-x-auto` 容器内部。下拉框使用 `position: absolute`，超出容器边界的部分被 `overflow` 裁剪。

**团队讨论**:

- 设计师: 提议将下拉框移到 overflow 容器外部，作为独立区块显示在城市卡片行下方
- 工程师: 采纳。将 `addRef` 移至外层包裹 div，"+"按钮留在卡片行内（正常水平滚动），搜索框在卡片行下方独立渲染
- 找茬的: 这样下拉框出现后不会被任何 overflow 截断吗？ → 确认，外层 div 无 overflow 限制
- 测试员: 测试 5 城市 + 打开下拉框场景，确认无截断

**修复**: 重构 add-city UI 布局：
- "+"按钮留在 `overflow-x-auto` 容器内的卡片行末尾
- 搜索下拉框 render 在 `overflow-x-auto` 容器的外部（同级兄弟元素）
- `addRef` 移至外层 div，outside-click 关闭仍然正常工作

**修改文件**: `components/CompareContent.tsx`

**修复**: 添加随机城市按钮，样式与其他页面一致。使用 `Object.values(CITY_SLUGS)` 获取所有有效 slug，随机跳转。
