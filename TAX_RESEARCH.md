## 2. 79 国/地区税收政策详查

### 分组逻辑

按税制复杂度分为 5 组：

- **A 组（免税/极简税制）** — 直接计算，几乎无误差
- **B 组（扁平税率）** — 单一税率 + 社保
- **C 组（标准累进制）** — OECD 标准累进 + 社保
- **D 组（复杂税制）** — 联邦/州/市多级税、特殊制度
- **E 组（数据受限）** — 制裁/动荡经济体、数据可信度低

## 3. 职业特殊税收待遇

### 3.1 医疗从业者

| 国家 | 适用职业 | 特殊政策 | 影响 |
|------|----------|----------|------|
| **法国** | 医生（自由职业） | 可选 BNC（非商业利润）制度 | 适用于开私人诊所的医生，不适用于受雇医生 |
| **印度** | 医生 | Section 44ADA 推定利润（收入≤₹75L）| 仅适用于个体执业，受雇者不受影响 |
| **日本** | 医生（开业） | 社会保险诊疗报酬 72% 概算经费 | 仅适用于独立诊所所有者 |

→ **结论**：我们的数据定位为「受雇者年薪」（employed salary），以上自由执业特殊制度**不适用**。

### 3.2 教师/学者

| 国家 | 适用职业 | 特殊政策 | 影响 |
|------|----------|----------|------|
| **中国** | 大学教授 | 稿酬所得 70% 计入；特许权使用费 80% 计入 | 仅适用于额外稿费/专利收入，基本工资无差异 |
| **美国** | 教师、大学教授 | IRS §135/529 教育储蓄免税；教师 $300 扣除 | 极小影响 |

→ **结论**：影响微乎其微，**不做特殊处理**。

### 3.3 公务员

| 国家 | 特殊政策 | 影响 |
|------|----------|------|
| **中国** | 公务员无需缴纳失业保险；公积金比例通常为 12%（上限） | 社保总扣除略低于企业员工 ~0.5% |
| **日本** | 共济组合（原公务员互助年金）→ 2015 年起并入厚生年金 | 现已无差异 |
| **韩国** | 公务员年金（政府年金）扣除率与国民年金不同 | 9%（略高于国民年金 4.5%） |

→ **结论**：中国和韩国公务员有**微小差异**，代码中可通过 职业=公务员 && 国家=中国/韩国 做 override。

### 3.4 IT/科技从业者（潜在与我们多个职业匹配）

| 国家 | 适用职业 | 特殊政策 | 有效期 | 影响 |
|------|----------|----------|--------|------|
| **罗马尼亚** | 软件工程师等 IT 岗 | IT 行业免个税（2024 年底已取消） | 已失效 | ❌ 不适用 |
| **印度** | 所有（SEZ 雇员） | 出口导向型 SEZ 企业 5 年免税 | 到期中 | 极少数人受益，**不处理** |

### 3.5 结论

**仅需对以下组合做代码级特殊处理**：
1. 🇨🇳 中国 × 公务员：社保中失业保险 0%（而非 0.5%），影响 <0.5%
2. 🇰🇷 韩国 × 公务员：公务员年金 9% 替代国民年金 4.5%，影响 ~4.5%

其余所有职业在所有国家统一适用标准税制。

## 4. 外来人口特殊税收政策

这是产品差异化的重要维度。我们的用户群体很可能是考虑海外就业/移居的人。

### 4.1 西班牙 — Beckham Law（贝克汉姆法案）

| 项目 | 详情 |
|------|------|
| 正式名称 | Régimen Especial para Trabajadores Desplazados（特殊劳动者制度） |
| 适用条件 | ① 前 5 年未在西班牙纳税；② 因工作合同搬迁到西班牙；③ 非自雇 |
| 核心优惠 | 所有收入按 **24% 固定税率**（而非 19-47% 累进），€600,000 以上部分 47% |
| 社保 | 与普通工人相同（~6.35%） |
| 有效期 | 搬迁年 + 5 年（共 6 个纳税年度） |
| 对我们的影响 | 大幅降低高收入者有效税率；巴塞罗那/马德里应提供此选项 |

→ **建议**：作为「外来人口模式」的 toggle 选项实现。

### 4.2 荷兰 — 30% Ruling

| 项目 | 详情 |
|------|------|
| 适用条件 | ① 从海外招聘；② 具有稀缺专长；③ 前 2 年中至少 16 个月未居住在荷兰 150km 范围内 |
| 核心优惠 | 应税收入的 **30% 免税**（实际上只对 70% 薪资征税） |
| 最低工资要求 | 2024 年：€46,107/年（30 岁以下科研人员 €35,048） |
| 有效期 | 最长 5 年（2024 年起逐步缩减：第 1-2 年 30%，第 3 年 20%，第 4-5 年 10%） |
| 对我们的影响 | 显著降低中高收入者有效税率 |

### 4.3 意大利 — 回归者减免（Impatriati Regime）

| 项目 | 详情 |
|------|------|
| 适用条件 | ① 前 2 年未在意大利纳税；② 在意大利至少工作 2 年 |

| 对我们的影响 | 大幅降低有效税率 |

## 6. 数据源口径交叉验证

### 6.1 验证方法论

对于口径不明确的地区（§1.3 中标为 🟡/🔴 的），采用以下多维验证策略：

#### 方法 A：多源对比

将我们的数据与至少 3 个独立数据源对比：

| 数据源 | 口径 | 覆盖范围 | 可信度 |
|--------|------|----------|--------|
| **OECD.Stat** Wages Database | 税前 gross，标准化 | 38 OECD 国 | 🟢 最高 |
| **ILO ILOSTAT** Global Wage Report | 税前名义 | 180+ 国 | 🟢 高 |
| **ERI/SalaryExpert** | 税前 gross | 全球 | 🟢 高（雇主验证） |
| **Glassdoor** | 用户填报，多数为税前 | 全球 | 🟡 中（口径不一） |
| **Numbeo** | 用户填报，"Average Monthly Net Salary (After Tax)" 明确标注税后 | 全球 | 🟡 中等（样本偏差） |
| **PayScale** | 税前 gross | 全球 | 🟡 中 |
| **Trading Economics** | 聚合官方统计 | 全球 | 🟢 高 |

#### 方法 B：税前/税后反推

1. 取 Numbeo 的 "Average Monthly Net Salary (After Tax)" 作为税后基准
2. 用我们的税率模型将我们的税前数据计算为税后
3. 如果计算税后与 Numbeo 税后偏差 < 15%，则验证通过
4. 偏差 > 25% 的城市需要人工复查

#### 方法 C：特定国家官方统计对比

| 国家 | 官方数据源 | 可获取指标 |
|------|-----------|-----------|
| 中国 | 国家统计局城镇私营/非私营单位就业人员年平均工资 | 税前 |
| 印度 | Income Tax Statistics (CBDT) | 各收入段纳税人数 |
| 泰国 | BOT Labour Statistics | 制造业/服务业平均工资 |
| 越南 | GSO Labour Force Survey | 平均月收入（含税） |

### 6.3 口径修正流程

如果验证发现某城市数据实为税后：

1. **不修改原始数据** — 保持 cities.json 为「声称的税前」值
2. 在 `taxData.ts` 中为该城市标记 `dataIsLikelyNet: true`
3. 前端税后模式下，若 `dataIsLikelyNet === true`，则不再重复扣税，直接显示原值
4. UI 上显示说明："该城市薪资数据可能已包含税后调整"

## 7. 计算模型设计

### 7.1 数据结构

```typescript
// lib/taxData.ts

/** 单个税阶 */
interface TaxBracket {
  upTo: number;     // 本币金额上限（Infinity = 最后一档）
  rate: number;     // 边际税率 0-1
}

/** 社保组件 */
interface SocialComponent {
  name: string;       // 如 "养老保险"、"SS"
  rate: number;       // 扣除比例
  monthCap?: number;  // 月基数上限（本币），超过部分不扣
  annualCap?: number; // 年扣除绝对上限（本币）
}

/** 国家税制 */
interface CountryTax {
  // 个税
  brackets: TaxBracket[];
  standardDeduction: number;  // 标准扣除/免征额（本币/年）
  additionalDeductions?: {    // 可选的额外扣除
    name: string;
    amount: number;
  }[];
  
  // 社保（雇员负担部分）
  social: SocialComponent[];
  
  // 货币
  currencyCode: string;
  currencyToUSD: number;
  
  // 外来人口优惠
  expatScheme?: {
    name: string;           // 如 "Beckham Law"
    type: "flat_rate" | "exemption_pct" | "no_social";
    flatRate?: number;      // 固定税率
    exemptionPct?: number;  // 免税比例
    incomeThreshold?: number; // 适用上限（USD）
  };
  
  // 数据质量
  confidence: "high" | "medium" | "low";
  dataIsLikelyNet?: boolean;  // 若原数据疑似已为税后
  lastUpdated: string;        // 如 "2025-01"
}

/** 城市级覆盖（仅在城市税率与国家不同时使用） */
interface CityTaxOverride {
  localTaxBrackets?: TaxBracket[];   // 州/市税阶
  localFlatRate?: number;             // 州/市扁平税率
  socialOverrides?: Partial<Record<string, SocialComponent>>;  // 如公积金比例
}
```

### 7.2 计算函数

```typescript
// lib/taxUtils.ts

export function computeNetIncome(
  grossUSD: number,
  country: string,
  cityId: number,
  options?: {
    profession?: string;     // 职业（公务员等特殊处理）
    isExpat?: boolean;       // 外来人口模式
  }
): {
  netUSD: number;
  effectiveRate: number;     // 综合有效税率（含社保）
  breakdown: {
    grossLocal: number;
    socialDeductions: number;
    incomeTax: number;
    localTax: number;
    netLocal: number;
  };
  confidence: "high" | "medium" | "low";
}
```

### 7.3 计算步骤

```
输入：grossUSD, country, cityId, profession, isExpat

1. 查找 CountryTax[country]
2. 若 dataIsLikelyNet === true → 直接返回 grossUSD（不重复扣税）
3. grossLocal = grossUSD / currencyToUSD

4. 社保扣除：
   for each SocialComponent:
     - 月基数 = min(grossLocal/12, monthCap)
     - 年扣除 = min(月基数 * rate * 12, annualCap)
   - 若 isExpat && expatScheme.type === "no_social" → 社保 = 0
   - 城市级覆盖：CityTaxOverride[cityId].socialOverrides
   - 职业覆盖：如韩国公务员年金

5. 应纳税所得额：
   taxableLocal = grossLocal - socialDeductions - standardDeduction - additionalDeductions

6. 累进个税：
   若 isExpat && expatScheme.type === "flat_rate":
     incomeTax = taxableLocal * flatRate
   若 isExpat && expatScheme.type === "exemption_pct":
     incomeTax = 对 taxableLocal * (1 - exemptionPct) 进行累进计算
   否则:
     incomeTax = 累进计算(taxableLocal, brackets)

7. 地方税（若有 CityTaxOverride）：
   若 localTaxBrackets: 对 taxableLocal 累进计算
   若 localFlatRate: taxableLocal * localFlatRate

8. 净收入：
   netLocal = grossLocal - socialDeductions - incomeTax - localTax
   netUSD = netLocal * currencyToUSD
   effectiveRate = 1 - netUSD / grossUSD
```

### 7.4 已知简化与局限

| 简化项 | 影响 | 说明 |
|--------|------|------|
| 忽略已婚/受抚养人扣除 | 中等 | 假设单身无子女为基准情景 |
| 忽略专项附加扣除（中国住房贷款利息等） | 小 | 最多降低有效税率 1-3% |
| 忽略年终奖单独计税（中国） | 中等 | 中国年终奖可选单独按月计税，但我们的数据为年总收入 |
| 忽略税收抵免（美国 EITC 等） | 低收入影响大 | 我们的 26 职业均非极低收入 |
| 忽略资本利得/投资收入 | 无 | 我们只处理薪资收入 |
| 社保上限使用全国统一值 | 小 | 部分国家按地区调整上限 |
| 汇率固定不更新 | 中等 | 与 cities.json 使用相同汇率基准 |

## 8. UI/界面设计方案

### 方案 A：导航栏 Toggle Pill（最小侵入）

```
┌─────────────────────────────────────────────────────┐
│ 🏠 首页  📊 排名  🎲 随机  │  职业 ▼  消费 ▼  语言 ▼  货币 ▼  [税前|税后]  🌙 │
└─────────────────────────────────────────────────────┘
```

- 在现有导航栏的 darkMode 按钮左侧增加一个与现有按钮风格一致的 toggle pill
- 默认显示「税前」，点击切换为「税后」
- 切换后所有收入相关数字瞬间更新（含排名、储蓄、购房年限等）
- 样式：与现有按钮一致的 `text-xs px-2 py-1 rounded border` 风格
- 外来人口优惠：仅在适用国家的城市详情页中，「税后」模式激活时，在导航栏下方出现一条小横幅：`📋 该城市支持外来人口税收优惠（Beckham Law）[启用]`

**优点**：改动最小，与现有设计完全一致，所有页面立即生效
**缺点**：导航栏空间进一步紧张（移动端可能换行）

### 方案 B：收入卡片内嵌切换 + 联动

```
┌─────────────────────────────────────────────────────┐

│ ┌──────────────────────────────┐                    │

│ │    综合税率 ~33.2%             │                    │
│ └──────────────────────────────┘                    │
└─────────────────────────────────────────────────────┘
```

- 仅在城市详情页的收入卡片下方加一行 radio 切换
- 切换后，该城市的收入/储蓄/排名等联动更新
- 其他页面（首页、排名页、对比页）保持税前
- 外来人口优惠作为第三个 radio 选项，仅在适用国家显示

**优点**：不影响导航栏；明确哪些数据受税影响
**缺点**：仅城市详情页可用，排名/对比无法按税后排序

### 方案 C：全局偏好 + 数据卡片标注

```
┌─────────────────────────────────────────────────────┐
│ 🏠 首页  📊 排名  🎲 随机 │ 职业 ▼  消费 ▼ 语言 ▼ 货币 ▼ 💰 ▼  🌙 │

                                          ↑
                                    点击展开下拉菜单：
                                    ┌──────────────────┐
                                    │ ○ 税前收入（默认）  │
                                    │ ○ 税后收入         │
                                    │ ○ 外来人口税后     │
                                    │────────────────── │
                                    │ ℹ️ 税后为简化估算    │
                                    └──────────────────┘
```

- 在导航栏增加一个 💰 图标按钮，点击展开下拉选择收入模式
- 全局生效（所有页面）
- 下拉菜单与现有的 职业/消费/语言/货币 下拉菜单风格一致（`<select>` 或自定义 dropdown）
- 选中「外来人口税后」时，非适用国家的城市自动 fallback 到普通税后
- 每张受影响的数据卡片右上角显示一个小标签 `税后` 或 `NET`

**优点**：三种模式优雅收纳，不增加导航栏宽度；全局生效
**缺点**：💰 图标含义不够直观，需要 tooltip；比方案 A 多一次点击

| 实现难度 | 简单 | 中等 | 中等 |

**建议**：方案 C 最均衡——全局生效、三种模式收纳整洁、移动端友好。

## 10. 风险与边界情况

### 10.1 已知风险

| 风险 | 严重度 | 缓解措施 |
|------|--------|----------|
| 各国税法每年修改 | 高 | 文件中标注 lastUpdated 字段；年度维护计划 |
| 用户误以为计算结果为精确值 | 高 | 强制显示 "简化估算" 标注 + 免责声明页 |
| 高通胀国家汇率波动 | 中 | 阿根廷、土耳其、黎巴嫩等使用 PPP 调整或加警示标注 |
| 某些国家我们的数据实为税后 | 中 | `dataIsLikelyNet` 标记避免重复扣税 |
| 公积金是否算"税"存在争议 | 低 | UI 明确标注"含社保/公积金扣除" |

### 10.2 边界情况

| 情况 | 处理方式 |
|------|----------|
| 税后计算后净收入为负 | 显示 "—"，不影响排名 |
| 某国无税率数据 | fallback 显示税前值 + 标注 "税后数据不可用" |
| 社保有月基数上限，高收入者实际扣除率更低 | 正确实现 monthCap 逻辑 |
| 阿根廷高通胀导致税阶实际上每季度调整 | 使用最新可得年度税阶 + 标注 |
| 黎巴嫩多重汇率（官方 vs 市场） | 使用市场汇率 + 标注 |
